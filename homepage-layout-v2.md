## 二、排版策略 — "排版驱动，不靠装饰"

### 核心原则
| 原则 | 做法 |
|:----|:-----|
| **留白即奢侈** | 每个区块之间 ≥48px 间距，呼吸感 |
| **文字即图形** | 标题用大号粗体日本字体（Noto Sans JP），视觉上就是图形 |
| **不对称节奏** | 不居中对称，采用"左上→右上→左下→右下"的 Z 字流动 |
| **icon 为锚点** | 4 个功能各配 1 个 emoji/icon，大小 ≥40px，作为视觉落点 |
| **无卡片感** | 不用矩形卡片边界，用文字+icon+微距组合代替 |

### 字号层级
```
日本語コーナー          → 40-48px, fw:900, 居左 or 居中
副标题 tagline           → 14-16px, fw:300, color: tertiary
功能标题                 → 18-20px, fw:700, icon旁
功能描述                 → 12-14px, fw:400, color: secondary
搜索栏输入文字           → 16px, 浅色 placeholder
游戏区副标题             → 14px, 带微标签
```

## 三、4功能详细布局方案

### 功能1：🔍 搜索（顶部贯穿式）

```
                   日本語コーナー
               ✦ 行小之的外语角 ✦

  ┌──────────────────────────────────┐
  │ 🔍  搜单词、搜句子、试试「美味しい」 │  [搜索]
  └──────────────────────────────────┘
```

**设计要点：**
- 全宽贯穿，放在品牌标题正下方
- 输入框背景：`bg-surface-2`，圆角 `radius-lg(14px)`
- 左侧🔍放大镜 icon（amber色）
- 右侧"[搜索]"按钮为品牌色渐变（amber→coral）
- placeholder 随机轮播保留
- 搜索结果下拉悬浮，半透明背景

**为什么放第一位：** 搜索是工具性功能，用户进入首页第一个动作通常就是搜索。放顶部符合心智模型。

---

### 功能2+3：📸 识图 + 📚 自动背单词 (Compact Pair)

```
           ┌────────────┐  ┌────────────┐
           │    📸      │  │    📚      │
           │  拍照识图    │  │  自动背单词  │
           │  相机识别日语 │  │  听力输入法  │
           └────────────┘  └────────────┘
           
            ← 左偏 12px →     ← 右偏 12px →
```

**设计要点：**
- **非对称排列**：两个方块大小一致，但在水平位置上左右微偏（各偏移12px），打破完美对称的呆板感
- **icon 上置**：icon 在方块中上位置，40-48px，带微发光
- **标题+描述**：icon 下方
- **悬停微交互**：hover 时 icon 微微上浮（translateY(-4px)）+ 底部出现 amber 下划线
- **点击区域**：整个方块可点击
- **颜色区分**：
  - 识图 → teal 氛围（相机/扫描绿调）
  - 自动背单词 → coral 氛围（听觉/录音粉调）

**为什么 pair 在一起：** 这两个功能是"输入"类型（一个视觉输入、一个听觉输入），属性相近，视觉上配对合理。

---

### 功能4：🎮 单词大冒险（底部宽区）

```
  ┌──────────────────────────────────┐
  │  🎮  单词大冒险                    │
  │  爬塔答题 · 拳击对战 · 趣味闯关    │
  │                                    │
  │  [🔥 今日挑战]  [🏆 排行榜]       │
  └──────────────────────────────────┘
```

**设计要点：**
- **最大视觉重量**：宽度为上面 pair 的两倍，高度约 1.5 倍
- **微辐标签**：内部增加"🔥 今日挑战""🏆 排行榜"两个子入口标签
- **背景氛围**：右侧淡淡的紫色/amber 渐变光晕
- **悬停交互**：整体上浮（translateY(-6px)）+ 边框发光
- **可以是轻微不规则形状**— 右上角切角/圆角不对称，增加设计感

**为什么放底部最宽：** 游戏是"输出/互动"型功能，是用户完成输入后的转化环节。放在底部视觉重量最大，暗示"这里是终点站"。

## 四、品牌标题区域设计

```
                   日本語コーナー
               ✦ 行小之的外语角 ✦
```

- **日本語コーナー**：Noto Sans JP, 40-48px, fw:900
  - 混排的片假名（コーナー）+汉字（日本語）本身就有设计感
  - 颜色：白色渐变，从纯白到 amber 微透
  - 可带极微的 `text-shadow` 或者 `letter-spacing: 2px`
  - 位置：居中或偏左（建议居中，配合下方搜索栏）
  
- **副标题**：14px, fw:300, 颜色 `text-tertiary`
  - 极细字重，与主标题形成对比
  - 可选带 `✦` 符号作为装饰
  - 不设 `border` 或背景

- **去掉**：🔥 学习天数 badge、开始学习/继续学习 CTA 按钮
  - 这些是"非4功能"元素，全部移除

## 五、整体间距与节奏

```
  ┌──────────────────────────────────┐
  │    ↓ 72px                         │
  │   日本語コーナー                    │
  │   ✦ 行小之的外语角 ✦              │
  │    ↓ 48px                         │
  │  ┌──── 搜索栏 ──────┐             │
  │  └──────────────────┘             │
  │    ↓ 56px                         │
  │      ┌──┐    ┌──┐                │
  │      │识│    │自│                │
  │      │图│    │动│                │
  │      └──┘    └──┘                │
  │    ↓ 40px                         │
  │  ┌──── 单词大冒险 ────┐           │
  │  └────────────────────┘           │
  │    ↓ 48px                         │
  │  [底部导航]                        │
  └──────────────────────────────────┘
```

间距全部用 8px 网格体系（已有 CSS token），视觉上向"奢侈留白"靠拢。

## 六、交互微动效

| 元素 | 交互 |
|:----|:-----|
| 搜索栏 | focus 时展开微扩 + amber 边框发光 |
| 识图 + 自动背单词 | hover → icon 上浮 4px + 底部 amber 下划线 |
| 单词大冒险 | hover → 整体上浮 6px + 边框渐变光晕 |
| 品牌标题 | 页面加载时标题 fade-in（0.6s ease） |
| 4功能 | 依次入场（stagger 0.15s），direction: from bottom 20px |

## 七、对比：现在 vs 改造后

| 当前首页 | 改造后首页 |
|:---------|:-----------|
| 10+ 区块，信息过载 | 5 个区块（title + 4功能） |
| 卡片堆叠，视觉同质 | 排版驱动，每个功能形态不同 |
| hero 区占 1/3 屏 | hero 只留标题字 |
| 功能平铺无大小区分 | 功能有大小层级（pair vs wide） |
| 装饰依赖背景图+光晕 | 装饰依赖留白+排版 |
| 不对称性 = 0（全居中对齐） | 不对称性：pair 偏移 + 大小对比 |
